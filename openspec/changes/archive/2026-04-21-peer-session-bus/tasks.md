## 3. Session registry + per-session mutex + timing-safe authentication

- [x] 3.1 Create `src/core/session-registry.ts`. Internal state uses `Map<string, SessionEntry>` — never a plain object. Export `PEER_BUS_MAX_UNREAD = 10000`. Export `SessionRegistry` class with methods `register(name, priorSessionToken?): { entry, rawToken }`, `authenticate(rawToken): SessionEntry | null`, `get(name)`, `addUnread(name, messageId)` (returns `false` if mailbox would exceed cap, `true` on success), `drainUnread(name, messageLookup, maxBytes): { messages, hasMore }`, `touch(name)`, `withLock(name, fn)`, `withLocks(nameA, nameB, fn)` (acquires lexicographically), `persist(): Promise<void>`, `load(path): Promise<void>`. Accept a `Logger` at construction.
- [x] 3.2 Implement `register` per the spec's three cases: (a) fresh name (no entry, `priorSessionToken` must be absent); (b) existing entry with non-empty `tokenHash` (`priorSessionToken` required, verified via SHA-256 + `timingSafeEqual`); (c) existing entry with empty `tokenHash` (`priorSessionToken` must be absent; populate `tokenHash`, preserve unread). Generate tokens via `crypto.randomBytes(32).toString('base64url')`. Return both entry and raw token; the raw token is returned only on successful calls. Redact tokens in all log output.
- [x] 3.3 Implement `authenticate` with constant-time iteration: compute `h = sha256(presentedToken)` once; iterate every entry; for each entry compute `timingSafeEqual(h, entry.tokenHash || ZERO_SENTINEL)` where `ZERO_SENTINEL = Buffer.alloc(32, 0)`; accumulate the matching entry into a local variable; DO NOT `return` or `break` inside the loop; branch only after the loop completes. Unit tests: valid token returns entry; empty-`tokenHash` entry never matches any presented token; 100-entry registry with the matching entry at position 99 completes in within ±10% of the wall-clock cost when it is at position 0 (use a statistical timing test with many samples); an entry with non-empty `tokenHash` that doesn't match never short-circuits the iteration.
- [x] 3.4 Implement a per-session async-mutex map. `withLock(name, fn)` serialises all calls for `name`. `withLocks(a, b, fn)` sorts the pair lexicographically and acquires in that order. Unit test for deadlock resistance: concurrent `send a → b` and `send b → a` both complete without hanging.
- [x] 3.5 Implement `persist()` via temp-file-rename: write full JSON state (iterating the `Map` to a plain `sessions` object at serialisation time) to `<path>.tmp`, then `fs.renameSync` over `<path>`. Never leave the live file partially written.
- [x] 3.6 Implement `load(path)` to parse `registry.json` if present. Build the in-memory `Map` by iterating `Object.entries(parsed.sessions ?? {})` — the `Map` itself resists prototype pollution, but also explicitly skip any key that fails the session-name regex and log with the key JSON-escaped. Clear `tokenHash` on every loaded entry. Validate timestamps parse via `Date.parse()`.
- [x] 3.7 Unit tests in `tests/session-registry.test.ts`: register fresh → entry + token; register against existing with correct `priorSessionToken` → rotates token; register against existing without `priorSessionToken` → error; register against empty-tokenHash entry without `priorSessionToken` → succeeds; `addUnread` returns `false` at cap; `drainUnread` stops at `maxBytes` and returns `hasMore: true` with remaining ids preserved in order; concurrent register-same-name serialises (second acquires lock, observes rotated-token state, succeeds with rotated `priorSessionToken`); persist-then-load round-trip preserves fields except `tokenHash`; load rejects `__proto__` key without polluting `Object.prototype` (explicit assertion); load rejects invalid-name entries with JSON-escaped log lines (no extra lines).
